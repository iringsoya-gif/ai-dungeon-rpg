슬라이드 1 — 타이틀 (35초)

안녕하세요. 202212577 박성안입니다.

제가 발표할 프로젝트 이름은 AI Dungeon RPG입니다.

지금 이 순간에도 실제로 접속이 가능한 라이브 서비스입니다. AI가 게임마스터가 되어 실시간으로 이야기를 만들어가는 한국어 웹소설 RPG인데요, 단순한 챗봇이 아니라 레벨업, 인벤토리, 퀘스트가 있는 진짜 RPG입니다.

슬라이드 2 — 문제 (55초)

ChatGPT로 RPG를 해보신 분 계신가요?

해보셨다면 금방 이 상황에 부딪히셨을 겁니다. 매번 "내 캐릭터는 전사고, HP는 80이고, 단검을 갖고 있고, 지금 던전 3층이야"를 직접 설명해줘야 합니다. 새로고침하면 다 사라지고, 대화가 길어지면 앞 내용을 잊어버립니다.

기존 텍스트 RPG 게임들은 미리 짜인 분기만 따라가고, AI 챗봇은 게임 시스템이 없습니다. 한국어로 시키면 갑자기 한자나 영어를 섞어 쓰는 문제도 있고요.

이 세 가지 문제를 한꺼번에 해결하고 싶었습니다.

슬라이드 3 — 솔루션 개요 (40초)

그래서 만든 것이 이 서비스입니다.

핵심은 세 가지입니다.

첫째, 완전 생성형 스토리. 플레이어의 모든 행동에 AI가 즉흥으로 반응합니다.

둘째, 풀 RPG 시스템. 레벨업, 인벤토리, 퀘스트, NPC 관계가 서버에서 자동으로 관리됩니다.

셋째, 장르별 한국어 문체. 판타지, SF, 공포, 현대 각각에 맞는 웹소설 스타일로 서술합니다.

---

슬라이드 4 — 시스템 아키텍처 (50초)

기술 구조를 간단히 설명드리겠습니다.

프론트엔드는 React로 만든 SPA로 Vercel에 배포했습니다. 게임 상태는 Zustand로 관리하고, AI 응답은 SSE로 실시간 수신합니다. BGM과 효과음은 Web Audio API로 외부 파일 없이 브라우저 안에서 직접 생성합니다.

백엔드는 FastAPI 서버로 Railway에 Docker 컨테이너로 배포했습니다. Google OAuth로 로그인, Polar.sh로 결제를 처리합니다.

AI는 Groq API를 통해 Meta의 LLaMA 3.3 70B 모델을 사용합니다. Groq 무료 티어 덕분에 추가 비용 없이 고성능 모델을 활용했습니다.

---

슬라이드 5 — AI GM 핵심 기술 (50초)

이 서비스의 핵심은 AI 게임마스터 설계입니다.

단순히 "RPG 진행해줘"라고 하면 모델은 들쭉날쭉한 결과를 냅니다. 그래서 시스템 프롬프트를 GM의 규칙서처럼 설계했습니다.

네 가지가 핵심입니다.

언어 규칙, 틀린 예시와 올바른 예시를 나란히 제시해 한자, 영어 혼입을 방지합니다.

장면 연속성, 직전 GM 서술의 장소에서 이어받도록 강제합니다.

Few-shot 문체, 장르별 예시 대화를 주입해 웹소설 스타일을 유지합니다.

구조화된 출력, GM 응답 마지막에 반드시 JSON 블록이 오도록 형식을 지정합니다. 이 JSON에서 HP 변화, 아이템, 퀘스트를 추출해 서버가 자동으로 처리합니다.

지금까지 설명한 내용을 직접 보여드리겠습니다.

▶ 영상 재생

보셨듯이, 플레이어가 행동을 입력하면 AI가 전투 장면을 즉석에서 서술하고, 서버가 결과를 파싱해서 HP와 XP를 자동으로 갱신합니다.

슬라이드 7 — 게임 시스템 (45초)

게임 시스템을 간단히 정리하겠습니다.

직업은 세계관에 맞게 20종이 제공됩니다. 판타지면 전사, 마법사, SF면 해커, 엔지니어 같은 식으로요. 직업마다 HP, MP, 힘, 지능, 민첩, 카리스마가 다르게 시작합니다.

전투에서 경험치를 쌓으면 레벨업이 되고 전 스탯이 자동으로 증가합니다. 행동 유형별 경험치 가이드를 GM에게 명시해서 밸런스 인플레이션을 방지했습니다.

특이한 점은 하드코어 모드입니다. 이 모드에서 HP가 0이 되면 캐릭터가 영구 사망합니다. 서버가 HP를 직접 감지하기 때문에 GM이 규칙을 어길 수 없습니다.

슬라이드 8 — Phase 2 개선 (40초)

초기 버전 출시 후 두 번째 단계에서 UX와 AI 품질을 대폭 개선했습니다.

UX 쪽에서는 GM 텍스트가 타이프라이터처럼 흘러나오는 효과를 requestAnimationFrame으로 구현했고, HP, XP 변화 시 색상 플래시 애니메이션을 추가했습니다. 상태 패널을 상태, 퀘스트, NPC, 지도 4탭으로 확장했고, 전투, 레벨업, 아이템, 오류마다 다른 효과음이 납니다.

완료된 모험을 공유 링크로 공개하는 모험 갤러리 페이지도 추가했습니다.

슬라이드 9 — 기존 챗봇 vs AI Dungeon RPG (45초)

표로 차이점을 정리하겠습니다.

가장 큰 차이는 상태의 지속성입니다. ChatGPT는 사용자가 직접 "내 HP는 50이야"라고 알려줘야 하지만, 이 서비스는 GM 응답에서 상태 변화를 파싱해 DB에 자동 저장합니다.

장기 플레이 맥락도 다릅니다. 일반 챗봇은 컨텍스트 윈도우를 넘으면 앞 내용을 잊지만, 자동 요약 압축으로 해결했습니다.

하드코어 모드, 장면 반응 BGM과 SFX, 모험 갤러리는 일반 챗봇에는 없는 부분입니다.

슬라이드 10 — 기술적 도전과 해결 (50초)

개발 과정에서 세 가지 큰 문제를 만났습니다.

첫째, 한자, 영어 혼입 문제입니다. 모델이 한국어 서술 중에 한자나 영어를 섞어 씁니다. 장르별 프롬프트에 나쁜 예와 좋은 예를 나란히 제시했고, 후처리 sanitizer로 걸러냅니다.

둘째, JSON 스트리밍 노출 문제입니다. AI가 상태 변화를 JSON으로 출력하는데, 스트리밍 중에 이 JSON이 그대로 화면에 보였습니다. 백틱 JSON 시작을 감지해서 그 시점부터 잘라내는 방식으로 해결했습니다.

셋째, 컨텍스트 폭증 문제입니다. 8천 토큰을 넘으면 오래된 기록을 AI로 요약하고, 최근 10턴과 요약만 전달하는 슬라이딩 윈도우를 구현했습니다.

슬라이드 11 — 수업 내용 연계 (40초)

이 프로젝트가 수업에서 배운 내용과 어떻게 연결되는지 정리했습니다.

LLM과 프롬프트 엔지니어링 관점에서, 모델이 토큰 단위로 텍스트를 생성한다는 원리를 이해하고, Few-shot 기법으로 출력 형식과 문체를 제어하는 것을 직접 구현했습니다.

API와 비동기 처리 관점에서는 FastAPI 백엔드 전체 구축과 async, await 기반 스트리밍을 구현했습니다.

데이터 구조와 상태 관리 관점에서는, 캐릭터 상태를 JSON으로 직렬화해 DB에 저장하고, 컨텍스트 관리에 슬라이딩 윈도우 알고리즘을 응용했습니다.

예상 질문 답변

Q. Groq 무료 티어 한계는?
분당 요청 수 제한과 일일 토큰 한도가 있습니다. 소규모 데모와 과제 수준에서는 충분하지만, 상용이라면 유료 전환이나 자체 모델이 필요합니다.

Q. SQLite를 쓴 이유는?
Railway 무료 티어에서 외부 DB 비용을 줄이기 위해서입니다. SQLAlchemy ORM을 쓰기 때문에 URL 한 줄만 바꾸면 PostgreSQL로 마이그레이션이 가능합니다.

Q. GM이 하드코어 규칙을 어길 수 있지 않나?
GM이 game_over를 false로 반환해도, 서버가 캐릭터 HP를 직접 확인해서 0이면 강제로 사망 처리합니다. 서버 로직이 GM 판단보다 우선합니다.

Q. 컨텍스트 압축 시 중요한 정보를 잃지 않나?
캐릭터 상태, 현재 위치, NPC 목록은 character_json과 시스템 프롬프트에 항상 포함되어 보존됩니다. 서사적 세부 내용은 일부 손실될 수 있습니다.

Q. BGM이 실제 음악처럼 들리나?
Web Audio API 오실레이터 기반이라 화음 드론 형태의 앰비언트 사운드입니다. 실제 음악 파일을 원한다면 TRACKS 객체에 MP3 URL을 지정하면 즉시 전환됩니다.
